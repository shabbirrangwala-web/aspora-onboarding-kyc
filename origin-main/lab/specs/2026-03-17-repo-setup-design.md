# Repo setup — design spec

## Date
2026-03-17

## Goal
Set up the Origin design system repo with proper structure, git history, GitHub remote under Vance-Club, and migrated planning docs. First-pass README for anyone landing on the repo.

## Decisions

- Repo lives at `Vance-Club/origin` (private), default branch `main`
- Code is the source of truth, Figma is synced to
- No component source code — implementations live in consuming projects
- Lab has four documentation areas: `thinking/` (strategic), `research/` (reference), `specs/` (design), `plans/` (implementation)
- Dating: specs/plans/research use `YYYY-MM-DD-` filename prefix. Thinking docs use `Last updated:` header.

## Directory structure

```
origin/
  foundations/                ← source of truth (tokens.json + guidelines.md per foundation)
    color/
    typography/
    spacing/
    motion/
    iconography/
    photography/
    voice/
    illustration/
  components/                 ← usage guidelines + uSpec/Figma links (no source code)
    product/                    (functional UI — buttons, inputs, selectors)
    editorial/                  (marketing/content — heroes, feature sections)
  outputs/                    ← generated by build, never hand-edited
    json/                       (canonical token data — what native engineers reference)
    css/                        (custom properties for web consumers)
    tailwind/                   (theme config for Tailwind projects)
    ai/                         (machine-readable design system context for LLMs)
  strategy/                   ← brand strategy content (pillars, positioning, personas)
  lab/                  ← exploration, generation, and documentation
    README.md                   (explains lab structure and conventions)
    thinking/                   (strategic docs — architecture, plans, parking lot)
    research/                   (audits, investigations, reference material)
    specs/                      (claude's design specs — what & why)
    plans/                      (claude's implementation plans — how & when)
    pipelines/                  (token generation — color OKLCH, future type/spacing)
    figma-plugin/               (consolidated plugin for all foundations)
    viewer/                     (lightweight token viewer/workbench)
  migration/                  ← old → new token mapping (temporary, delete post-rebrand)
  README.md
  CLAUDE.md
  package.json
```

Directories for future workstreams (`pipelines/`, `figma-plugin/`, `viewer/`, all `foundations/` subdirs, `components/`, `outputs/`, `strategy/`, `migration/`) get `.gitkeep` placeholders only.

## Deliverables

1. **Directory scaffold** — all directories above, with `.gitkeep` where empty
2. **Migrate planning docs** — move root-level files to lab:
   - `current-working-plan-v1.md` → `lab/thinking/`
   - `design-system-architecture.md` → `lab/thinking/`
   - `for-later.md` → `lab/thinking/`
   - `ai-consumption-research.md` → `lab/research/2026-03-11-ai-consumption-research.md`
   - `native-app-token-audit.md` → `lab/research/2026-03-17-native-app-token-audit.md`
3. **lab/README.md** — documents the lab structure and dating conventions
4. **README.md** — first pass covering: what Origin is, who it's for, repo structure overview, three-repo architecture (origin, docs site, brand-lab), current status
5. **CLAUDE.md** — project context for Claude Code sessions
6. **package.json** — minimal (`name`, `description`, `private: true`)
7. **Git init + first commit** on `main`
8. **GitHub remote** — create `Vance-Club/origin` (private), push

## Done when

- `Vance-Club/origin` exists and is accessible on GitHub
- All directories exist with `.gitkeep` where empty
- Thinking docs are in `lab/thinking/`, research in `lab/research/`
- Specs and plans are in `lab/specs/` and `lab/plans/`
- `lab/README.md` documents the structure and conventions
- README renders correctly on GitHub
- `.claude/` directory is gitignored
