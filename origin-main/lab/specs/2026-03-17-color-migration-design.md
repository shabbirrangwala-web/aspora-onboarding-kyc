# Color-theory migration — design spec

## Date
2026-03-17

## Goal
Migrate the color-theory project into Origin. Pipeline, tokens, Figma plugin, and ramp builder land in their proper locations. Token format converts from TypeScript to DTCG JSON. Pipeline and Figma plugin decouple through `foundations/color/` as the interface.

## Source project
`/Users/hakeeb/Documents/Build/color-theory/scripts/figma-plugin/`

## Decisions

- Token format: W3C DTCG (Design Tokens Community Group)
- Three token files: `primitives.json`, `semantic.light.json`, `semantic.dark.json`
- Modes handled by file separation, not extensions
- Scopes are a Figma plugin concern, not a foundation concern
- Pipeline writes DTCG JSON directly (no intermediate TypeScript)
- Brand anchor hexes get their own input file
- Ramp builder HTML comes along as a dev tool
- Presentation deck and app mockups stay behind

## DTCG format

### primitives.json

```json
{
  "color": {
    "$type": "color",
    "neutral": {
      "50":  { "$value": "#ffffff" },
      "100": { "$value": "#f3f1f1" },
      ...
    },
    "maroon": {
      "50":  { "$value": "#fff8f7" },
      ...
      "800": { "$value": "#680d0d", "$description": "Brand anchor" },
      ...
    },
    "white-alpha": {
      "100": { "$value": "#ffffffff" },
      "70":  { "$value": "#ffffffb3" },
      ...
    }
  }
}
```

### semantic.light.json

```json
{
  "color": {
    "$type": "color",
    "surface": {
      "base":     { "$value": "{color.neutral.50}" },
      "raised":   { "$value": "{color.neutral.100}" },
      "sunken":   { "$value": "{color.neutral.200}" },
      "overlay":  { "$value": "{color.neutral.50}" },
      "contrast": { "$value": "{color.neutral.1000}" }
    },
    "on-surface": {
      "primary":   { "$value": "{color.neutral.1000}" },
      "secondary": { "$value": "{color.neutral.700}" },
      ...
    }
  }
}
```

### semantic.dark.json

Same structure, different primitive references.

### anchors.json

```json
{
  "neutral": { "hex": "#680D0D", "isNeutral": true, "neutralChromaMax": 0.012 },
  "maroon":  { "hex": "#680D0D", "step": 800 },
  "crimson": { "hex": "#B33056", "step": 600 },
  "peach":   { "hex": "#FFAE92", "step": 300 },
  "gold":    { "hex": "#FAA810", "step": 400 },
  "lime":    { "hex": "#AFEA6A", "step": 300 },
  "teal":    { "hex": "#097E8D", "step": 600 },
  "blue":    { "hex": "#4E68F4", "step": 600 }
}
```

Note: Alpha ramps (white-alpha, black-alpha) are generated by the pipeline independently — fixed opacity steps (100/70/40/16/8), not driven by anchors.

## Key refactors

1. **Pipeline output**: TypeScript → DTCG JSON. Generation logic (OKLCH math, Culori) unchanged. Only the serialization changes.
2. **Semantic token split**: Single `SEMANTIC_TOKENS` array → two DTCG files. Light/dark refs become separate files with same keys but different alias targets.
3. **Figma plugin imports JSON at build time**: Instead of importing `generated-colors.ts` and `tokens.ts`, plugin imports DTCG JSON from `foundations/color/` resolved by Vite at build time. `vite.config.main.ts` needs path alias to resolve `foundations/color/`.
4. **Scope extraction**: Scope arrays move from token definitions to `scope-config.ts` in the plugin. Grouped by token path pattern (e.g., `surface/*` → `ALL_FILLS`).

## Build scripts

### Pipeline (`lab/pipelines/color/`)
```
npm run generate    # Read anchors.json → OKLCH generation → write primitives.json
npm run verify      # Validate semantic files reference valid primitives
npm run build       # generate + verify
```

### Figma plugin (`lab/figma-plugin/`)
```
npm run build:ui    # Compile plugin UI
npm run build:main  # Compile plugin code
npm run build       # build:ui + build:main
```

## Done when

- `foundations/color/` has all four files (anchors, primitives, semantic light, semantic dark) in valid DTCG format
- `lab/pipelines/color/generate.ts` reads anchors and produces `primitives.json` matching the current generated values
- `lab/pipelines/color/verify.ts` validates all semantic references resolve
- `lab/figma-plugin/` builds and the plugin produces the same Figma variables as the current color-theory plugin
- `lab/viewer/color.html` loads and displays ramps correctly
- `guidelines.md` captures the key rationale from color-theory
- All committed and pushed

## What this does NOT cover

- Extending the Figma plugin for other foundations (typography, spacing)
- Output generation (CSS, Tailwind, JSON exports) — that's a separate workstream
- Updating the ramp builder HTML to read from DTCG instead of inline data
