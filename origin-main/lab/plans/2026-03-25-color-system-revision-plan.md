# Color system revision implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Introduce functional red/green/yellow ramps, restructure semantic tokens (solid/light/on-solid/on-light/border), add accent and overlay tokens, fix disabled states — with a browser-based review page for visual approval before committing token changes.

**Architecture:** Two-phase approach. Phase A generates new primitive ramps and builds a review page so the design system owner can see everything in context before any semantic tokens change. Phase B (gated on Phase A approval) updates the semantic token files, adds overlays and disabled tokens, and verifies the full pipeline.

**Tech Stack:** TypeScript (Culori for OKLCH), HTML/CSS/JS (review page), W3C DTCG JSON (tokens)

**Spec:** `lab/specs/2026-03-25-color-system-revision-design.md`

## Phase A: Generate ramps and build review page

### Task 1: Select anchor hexes for red, green, yellow

**Files:**
- Modify: `foundations/color/anchors.json`

The pipeline auto-detects which step an anchor belongs to based on OKLCH lightness. We want anchors that land at step 600 (L≈0.54) for red and green (so `solid` tokens are at the brand anchor), and step 500 or 600 for yellow (since pure yellow is naturally high-lightness).

- [ ] **Step 1: Research candidate anchors**

Use Culori to convert candidates to OKLCH and find which step they'd land at. Good starting points from established systems:

| Candidate | Hex | Source | Expected hue |
|-----------|-----|--------|-------------|
| Red A | `#DC2626` | Tailwind red-600 | ~25 |
| Red B | `#D32F2F` | Material red-700 | ~27 |
| Red C | `#E53935` | Material red-600 | ~26 |
| Green A | `#16A34A` | Tailwind green-600 | ~150 |
| Green B | `#2E7D32` | Material green-800 | ~148 |
| Green C | `#059669` | Tailwind emerald-600 | ~165 (too teal) |
| Yellow A | `#CA8A04` | Tailwind yellow-600 | ~88 |
| Yellow B | `#F59E0B` | Tailwind amber-500 | ~80 (too gold) |
| Yellow C | `#EAB308` | Tailwind yellow-500 | ~90 |

Run a quick script to convert each to OKLCH and print L/C/H and which step it would land at:

```typescript
import { parse, oklch } from "culori";
const candidates = {
  "red-A": "#DC2626", "red-B": "#D32F2F", "red-C": "#E53935",
  "green-A": "#16A34A", "green-B": "#2E7D32", "green-C": "#059669",
  "yellow-A": "#CA8A04", "yellow-B": "#F59E0B", "yellow-C": "#EAB308",
};
const LIGHTNESS_MAP = { 50:1.0, 100:0.96, 200:0.92, 300:0.85, 400:0.76, 500:0.65, 600:0.54, 700:0.43, 800:0.33, 900:0.23, 950:0.18, 975:0.15, 1000:0.12 };
for (const [name, hex] of Object.entries(candidates)) {
  const o = oklch(parse(hex)!);
  let best = 500, minD = Infinity;
  for (const [s, l] of Object.entries(LIGHTNESS_MAP)) {
    const d = Math.abs((o?.l ?? 0) - l);
    if (d < minD) { minD = d; best = +s; }
  }
  console.log(`${name}: ${hex} → L=${o?.l?.toFixed(3)} C=${o?.c?.toFixed(3)} H=${o?.h?.toFixed(1)} → step ${best}`);
}
```

Run: `cd lab/pipelines/color && npx tsx -e '<script above>'`

Pick the candidate per color that: (a) lands at step 600, (b) has the highest chroma, (c) has a hue in our target range.

## Phase B: Update semantic tokens (after review approval)

### Task 3: Update semantic.light.json

**Files:**
- Modify: `foundations/color/semantic.light.json`

- [ ] **Step 1: Restructure status groups and add accent**

Replace the 4 status groups (error, success, warning, info) with the new 5-token structure. Add accent. Add overlay tokens. Update interactive and disabled tokens.

The full target file should have these groups in this order:
1. `surface` — unchanged
2. `on-surface` — disabled value changes from neutral-300 to neutral-400
3. `border` — add `disabled` token (neutral-200)
4. `interactive` — keep primary + secondary, add disabled (neutral-200), remove hover/pressed/disabled variants
5. `error` — solid (red-600), light (red-50), on-solid (neutral-50), on-light (red-700), border (red-600)
6. `success` — solid (green-600), light (green-50), on-solid (neutral-50), on-light (green-700), border (green-600)
7. `warning` — solid (yellow-600), light (yellow-50), **on-solid (neutral-1000 — dark text, not white, because yellow is a light-value hue)**, on-light (yellow-800), border (yellow-600)
8. `accent` — solid (blue-600), light (blue-50), on-solid (neutral-50), on-light (blue-700), border (blue-600)
9. `on-brand` — unchanged
10. `overlay` — keep scrim (unchanged), add solid-hover (white-alpha-8), solid-pressed (white-alpha-16), light-hover (black-alpha-8), light-pressed (black-alpha-16)
11. `brand` — unchanged

**Critical:** `warning.on-solid` MUST be `neutral-1000` (dark text), not `neutral-50`. Yellow solid fills are too light for white text.
