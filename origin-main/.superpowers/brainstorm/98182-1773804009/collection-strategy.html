How should Origin organize Figma variable collections?

Collections are the top-level containers for variables in Figma. This decision affects how designers browse, how the plugin syncs, and how primitives vs semantics are separated.

A

One collection per foundation

Separate collections for Color, Typography, Spacing, Radius. Each contains both primitives and semantics. Primitives hidden from publishing.

📦 Collections:
├── Color
│   ├── primitives/ (hidden)
│   └── semantic/
├── Typography
│   ├── primitives/ (hidden)
│   └── semantic/
├── Spacing
│   ├── primitives/ (hidden)
│   └── semantic/
└── Radius
    ├── primitives/ (hidden)
    └── semantic/

Pros

  • Clean separation by domain
  • Each foundation evolves independently
  • Mirrors repo structure

Cons

  • 4+ collections to manage
  • Cross-collection aliases needed for semantic → primitive references
B

Two collections: Primitives + Semantics

All primitive values (color, type sizes, spacing, radius) in one collection. All semantic tokens in another. Primitives collection hidden from publishing.

📦 Collections:
├── Primitives (hidden from publishing)
│   ├── color/
│   ├── typography/
│   ├── spacing/
│   └── radius/
└── Semantics
    ├── color/
    ├── typography/ (+ text styles)
    ├── spacing/
    └── radius/

Pros

  • Only 2 collections total
  • Clear primitive/semantic boundary
  • One hide-from-publishing toggle

Cons

  • Large collections as system grows
  • Different foundations mixed in same container
C

Three collections: Primitives + Semantic Color + Semantic Layout

Primitives together (hidden). Semantic color separate (has light/dark modes). Semantic typography + spacing + radius together (no modes).

📦 Collections:
├── Primitives (hidden from publishing)
│   ├── color/
│   ├── typography/
│   ├── spacing/
│   └── radius/
├── Color (modes: light / dark)
│   ├── surface/
│   ├── on-surface/
│   └── ...
└── Layout
    ├── typography/ (+ text styles)
    ├── spacing/
    └── radius/

Pros

  • Color gets its own modes (light/dark)
  • Non-modal foundations grouped logically
  • Scales well when dark mode lands

Cons

  • 3 collections — middle ground complexity
  • "Layout" naming is debatable

Note: Typography text styles are separate from variables (Figma limitation). The plugin creates both: variables for primitive values, text styles for semantic composites.
