# Origin — Aspora design system

## What this is

Origin is the source of truth for Aspora's design decisions. It owns tokens, guidelines, and tooling. It does not contain component source code — platform teams build their own implementations.

## Repo structure

- `foundations/` — Token JSON + guidelines markdown per foundation (color, typography, spacing, motion, iconography, photography, voice, illustration)
- `components/` — Usage guidelines and links to Figma/uSpec specs. No source code. Split into `product/` (functional UI) and `editorial/` (marketing/content)
- `outputs/` — Generated by build, never hand-edited. JSON (canonical tokens), CSS (custom properties), Tailwind (theme config), AI (LLM-consumable context)
- `strategy/` — Brand strategy content (pillars, positioning, personas)
- `lab/` — Documentation, exploration tools, and generation pipelines. See `lab/README.md` for full structure
- `migration/` — Old-to-new token mapping. Temporary, deleted post-rebrand

## Lab documentation

- `lab/thinking/` — Strategic docs (architecture, plans, parking lot). Updated over time.
- `lab/research/` — Audits and reference material. Date-prefixed filenames.
- `lab/specs/` — Design specs (what & why). Date-prefixed. Written during brainstorming.
- `lab/specs/components/` — Component specs (anatomy, variants, tokens, usage). Plain names matching component. Written during component design, moved to `components/product/` or `components/editorial/` when finalized.
- `lab/plans/` — Implementation plans (how & when). Date-prefixed. Written from approved specs.

## Key decisions

1. Code is the source of truth, not Figma. Figma is synced to and contributed from.
2. No component source code in this repo. Implementations live in consuming projects.
3. Manual token consumption. Engineers reference `outputs/json/` and update platform files themselves. Foundations change rarely.
4. Two kinds of foundational truth: token-based (JSON, generated by pipelines) and prose-based (markdown IS the source of truth).
5. Three-repo architecture: origin (produces), aspora-design-site (docs consumer), aspora-brand-lab (studio consumer).
6. Single consolidated Figma plugin pushes all foundations, not one per foundation.
7. The rebrand IS the migration. No separate migration workstream.

## Conventions

- Foundation tokens use `foundations/<name>/primitives.json` for raw values and `foundations/<name>/semantic*.json` for intent-based tokens (e.g., `semantic.light.json` + `semantic.dark.json` for color)
- Typography has separate web and mobile files:
  - `foundations/typography/primitives.json` + `semantic.json` — mobile apps (iOS, Android). Root key: `"typography"`
  - `foundations/typography/web-primitives.json` + `web-semantic.json` — web interfaces (CSS, TS, Tailwind). Root key: `"web-typography"`
  - Web primitives start identical to mobile but will migrate to rem units. Web semantics reference `{web-typography.*}` tokens
  - The build script (`scripts/build-ts.mjs`) uses web-primitives.json + web-semantic.json for all generated web outputs
  - Mobile platform teams consume primitives.json + semantic.json directly; they do not use the build script outputs
- Foundation guidelines live in `foundations/<name>/guidelines.md`
- Outputs are generated from foundations — never edit files in `outputs/` directly
- Color uses OKLCH for perceptual uniformity. Anchors are brand hexes, ramps are generated.
- Warm neutrals derived from maroon hue, max chroma 0.012
- Sentence case everywhere — collection names, plugin names, labels, headings
- Never delete Figma artifacts (variables, styles, collections) — always update in place, flag orphans for manual review
- Case-insensitive matching when looking up Figma collections or variables by name — prevents duplicates from casing changes

## Native platform notes

- iOS and Android tokens are ~90% aligned (same naming, same values)
- Android has two view systems: Compose (token-aware) and XML (legacy, dying)
- Server-driven UI uses string identifiers (BODY_M_REGULAR etc.) — preserve these
- Dark mode infrastructure exists on both platforms but is unused. Phase 2.
